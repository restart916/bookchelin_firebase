# EPUB 목차 이동 스크롤 안정화 설계

## 목표

연속 스크롤 읽기는 유지하면서, 목차에서 중간 장으로 이동했을 때 이전 장이 위에 선로딩되며 현재 위치가 크게 바뀌는 현상을 제거한다. 장시간 독서 시에는 epub.js의 기존 뷰 정리 동작을 유지해 메모리가 책 전체 크기에 비례해 증가하지 않게 한다.

## 설계

- 목차 이동 직전에 continuous manager의 선로딩 거리(`offset`)를 0으로 낮춰 초기 `fill()`이 이전 장을 prepend하지 못하게 한다.
- 목표 장이 상단에서 기본 선로딩 거리 이상 멀어질 때 원래 offset을 복원한다. 따라서 아래 방향 연속 로딩은 기존처럼 미리 시작된다.
- 목차 이동 직후 위쪽 장으로 돌아가는 동작은 하단의 `이전` 버튼이 담당한다. 이전 장을 스크롤로 prepend하는 것보다 위치 안정성을 우선한다.
- epub.js가 직접 높이 보정을 수행하므로 `.epub-container`의 브라우저 스크롤 앵커링을 끈다.
- 목차 이동은 중복 클릭을 막고 `display()` 완료를 기다린다. 실패하더라도 잠금과 억제 플래그는 반드시 해제한다.
- `relocated` 이벤트에서는 앱 CFI 알림만 보낸다. 글꼴과 여백 규칙은 theme/content hook 경로로 새 장에도 적용되므로 위치 이벤트마다 레이아웃을 다시 계산하지 않는다.

## 테스트

- 순수 상태 도우미 테스트로 목차 이동 시작/종료 시 prepend 억제와 잠금 해제를 검증한다.
- manager 설정 도우미 테스트로 목차 이동 시 offset을 0으로 낮추고 안전 거리 이후 복원하는지 검증한다.
- 기존 단위 테스트, lint, production build를 실행한다.
- 실제 뷰어에서 중간 목차 이동 직후 목표 장이 유지되고 아래로 스크롤할 때 다음 장이 이어지는지 확인한다.

## 범위 밖

- epub.js 라이브러리 자체 수정 또는 포크
- EPUB 파일 구조 변경
- 이전/다음 버튼 제거
