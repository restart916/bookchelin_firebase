# 홈 상단 자동 캐러셀 + 수동 핀 설계

- 날짜: 2026-06-18
- 상태: 사용자 승인
- 범위: `bookchelin_firebase` Cloud Functions + Vue 2 어드민
- 모바일 앱 변경: 없음

## 1. 배경

홈에는 이미 `지금 인기`와 `오늘의 발견` 자동 행이 있다. 반면 최상단 캐러셀은 동적 홈을 지원하는 앱에서도 실질적으로 `main_books` 고정 풀을 날짜별로 10권씩 잘라 보여준다. 순서는 바뀌지만 후보군은 그대로라 장기간 같은 책이 반복된다.

현재 버전별 동작은 다음과 같다.

- 아주 오래된 앱은 `main_books`를 직접 읽는다.
- 동적 홈을 지원하는 Android/Flutter 앱은 `home_dynamic/current.carousel`을 우선 읽고, 문서가 없거나 비어 있으면 `main_books`로 폴백한다.
- 서버의 `generateHomeDynamic()`은 현재 `main_books.book_id`를 읽어 `home_dynamic/current.carousel`을 만든다.

따라서 Cloud Functions의 편성 로직만 바꾸면 모바일 앱을 새로 배포하지 않고도 이미 동적 홈을 지원하는 앱의 캐러셀을 바꿀 수 있다. `main_books`는 아주 오래된 앱을 위해 그대로 보존한다.

## 2. 목표와 비목표

### 목표

- 최상단 캐러셀에 매일 자동으로 선정한 책 5권을 노출한다.
- 자동 5권은 서로 다른 역할을 맡아 독서 시작과 카탈로그 발견을 함께 높인다.
- 운영자가 필요할 때 책을 수동 핀으로 추가하고 최종 노출 위치를 지정할 수 있게 한다.
- 수동 핀은 자동 5권을 대체하지 않고 추가된다.
- `지금 인기`, `오늘의 발견`, 수동 핀, 자동 캐러셀 사이의 중복을 제거한다.
- 모바일 앱을 배포하지 않고 Firebase Functions와 어드민 배포만으로 출시한다.

### 비목표

- 사용자별 이어읽기 첫 장과 취향 개인화는 이번 범위에서 제외한다.
- 캐러셀 카드에 선정 이유 문구를 표시하지 않는다. 현재 앱은 책 ID 배열만 읽으므로 문구 표시는 모바일 앱 변경이 필요하다.
- `main_books` 컬렉션을 삭제하거나 덮어쓰지 않는다.
- `지금 인기`와 `오늘의 발견`의 기존 선정 규칙을 재설계하지 않는다.
- `time_event` 기반 `NEW 추천 책 등록` 기능은 캐러셀과 무관하므로 변경하지 않는다.

## 3. 버전 호환 구조

```text
아주 오래된 앱
  └─ main_books 직접 조회

동적 홈 지원 앱
  └─ home_dynamic/current.carousel
       ├─ 활성 수동 핀 N권
       └─ 자동 선정 5권
       ※ 문서가 없거나 비면 앱에서 main_books 폴백
```

새 편성기는 더 이상 `main_books`를 캐러셀 후보나 제외 목록으로 사용하지 않는다. `main_books`는 구버전 전용 정적 데이터로 남는다.

함수가 실패하면 `home_dynamic/current`를 덮어쓰지 않아 직전 성공 편성이 유지된다.

## 4. 데이터 모델

### 4.1 `home_carousel_pins/{pinId}`

```json
{
  "book_id": "book-document-id",
  "position": 1,
  "is_active": true,
  "start_date": "2026-06-18",
  "end_date": null,
  "created_at": "<Timestamp>",
  "updated_at": "<Timestamp>"
}
```

- `position`은 최종 캐러셀의 1부터 시작하는 위치다.
- `start_date`와 `end_date`는 선택 필드인 KST `YYYY-MM-DD` 문자열이며 시작일과 종료일을 모두 포함한다.
- 활성 조건은 `is_active === true`, 시작일이 없거나 오늘 이하, 종료일이 없거나 오늘 이상인 경우다.
- 동일한 `book_id`의 활성 핀은 하나만 허용한다.
- 어드민은 동일 위치의 활성 핀 저장을 막는다.
- 서버는 잘못된 중복 위치나 중복 책이 있어도 결정적으로 한 번만 포함한다.

### 4.2 `home_dynamic/current`

기존 클라이언트 호환을 위해 저장 shape을 바꾸지 않는다.

```json
{
  "updated_at": "<Timestamp>",
  "date": "2026-06-18",
  "carousel": ["pinA", "auto1", "auto2", "pinB", "auto3", "auto4", "auto5"],
  "trending": [{"book_id": "...", "reader_count": 3}],
  "discover": ["..."]
}
```

### 4.3 `home_dynamic/_carousel_state`

자동책의 최근 캐러셀 노출을 추적한다.

```json
{
  "updated_at": "<Timestamp>",
  "day": 20622,
  "baseline": {
    "bookA": 20608
  },
  "last_shown_day_by_book": {
    "bookA": 20622,
    "bookB": 20621
  }
}
```

- `last_shown_day_by_book`은 현재 존재하는 책의 마지막 자동 캐러셀 노출일을 보존하고, 삭제된 책만 정리한다. 현재 카탈로그 규모에서는 단일 문서 한도 안에 충분히 들어간다.
- `baseline`은 그날 첫 실행 전 상태다. 같은 날 핀 변경 등으로 재실행할 때 이 기준을 재사용해 노출일이 이중 증가하지 않게 한다.
- 같은 KST 날짜에 재실행해도 동일한 핀과 후보 입력에서는 같은 결과가 나오도록 결정적 정렬을 사용한다.

## 5. 자동 5권 선정

### 5.1 공통 후보 조건

자동 후보는 다음 조건을 모두 만족해야 한다.

- `books/{bookId}`가 존재한다.
- `hidden === false`다.
- `image_url`이 비어 있지 않은 문자열이다.
- 오늘의 `trending`, `discover`, 활성 수동 핀에 포함되지 않는다.
- 최근 14일 이내 캐러셀에 자동책으로 노출되지 않았다.

14일 제한 때문에 특정 역할의 후보가 없으면 제한을 가장 오래전에 노출된 책부터 완화해 전체 5권을 채운다. 가시 책이 5권보다 적은 극단적 상황에서는 가능한 수만 기록한다.

### 5.2 역할별 슬롯

각 역할은 앞 역할에서 선택한 책을 제외한 뒤 한 권을 선택한다.

1. **최근 상승책**
   - 기존 `rankTrending()` 전체 랭킹에서 `지금 인기`로 뽑힌 책 다음의 최상위 후보를 선택한다.
   - 최근 독자 가중치와 독서시간을 활용하되 인기 행과 중복하지 않는다.

2. **꾸준히 읽히는 책**
   - 최근 집계 구간에서 총 독서시간이 높은 남은 책을 선택한다.
   - 동점은 고유 독자 수, 책 ID 순으로 결정한다.

3. **카테고리 대표책**
   - 가시 책에 실제로 존재하는 카테고리를 안정 정렬하고 KST 날짜 인덱스로 하나를 순환 선택한다.
   - 해당 카테고리에서 최근 노출이 가장 적고 `order`가 높은 책을 선택한다.

4. **재발견책**
   - 최근 집계 구간에 독서 기록이 없는 남은 책 중 마지막 캐러셀 노출이 가장 오래된 책을 선택한다.
   - 한 번도 노출되지 않은 책을 먼저 선택한다.

5. **탐색책**
   - 아직 선택되지 않은 적격 후보를 책 ID로 안정 정렬한 뒤 KST 날짜 인덱스로 결정적 순환 선택한다.
   - 완전 난수가 아니므로 같은 날 재실행 결과가 바뀌지 않고, 장기적으로 후보군을 고르게 순환한다.

역할별 후보가 없으면 남은 전체 적격 후보에서 `마지막 노출이 오래된 순 → book_id 순`으로 보충한다.

### 5.3 홈 전체 중복 우선순위

한 책은 홈 상단 세 영역 중 한 곳에만 노출한다. 우선순위는 다음과 같다.

```text
수동 핀 > 지금 인기 > 오늘의 발견 > 자동 캐러셀 5권
```

- 활성 핀은 `trending`과 `discover` 후보에서도 제외한다.
- `trending`으로 선택한 책은 `discover`와 자동 캐러셀에서 제외한다.
- `discover`로 선택한 책은 자동 캐러셀에서 제외한다.
- 자동 캐러셀은 마지막에 남은 후보에서 역할별로 선택한다.

## 6. 수동 핀과 최종 위치 조합

수동 핀은 자동 5권과 별개로 모두 추가한다.

예를 들어 활성 핀 두 개의 위치가 1과 4이면 최종 7권은 다음처럼 구성한다.

```text
1 핀 A
2 자동 1
3 자동 2
4 핀 B
5 자동 3
6 자동 4
7 자동 5
```

조합 규칙은 다음과 같다.

1. 활성 핀을 `position ASC`, `created_at ASC`, 문서 ID ASC로 정렬한다.
2. 최종 길이를 `활성 고유 핀 수 + 자동책 수`로 정한다.
3. 유효 범위는 `1..최종 길이`다. 범위를 벗어난 위치는 맨 뒤로 보정한다.
4. 지정 위치가 이미 다른 핀에 점유됐다면 다음 빈 위치에 배치하고, 뒤에도 빈자리가 없으면 앞쪽 첫 빈 위치에 배치한다.
5. 핀 위치를 배치한 뒤 나머지 빈 위치를 자동 5권의 역할 순서대로 채운다.
6. 핀 책은 자동 후보에서 먼저 제외해 최종 배열에 중복되지 않게 한다.

어드민 정상 경로에서는 위치 충돌과 범위 오류를 저장 전에 막고, 서버 보정은 직접 Firestore 수정이나 경합에 대한 방어로만 사용한다.

## 7. 생성 시점과 즉시 반영

- 기존 `daily_job`이 매일 전체 편성을 생성한다.
- `home_carousel_pins`의 생성·수정·삭제에는 v1 Firestore `onWrite` 트리거를 연결해 `generateHomeDynamic()`을 즉시 다시 실행한다.
- 프로젝트의 Cloud Functions는 v1 API를 유지하며 이 기능만 v2로 마이그레이션하지 않는다.
- 같은 날 여러 번 실행돼도 결정적 선정과 상태 처리로 자동 5권은 불필요하게 바뀌지 않는다.
- 동시 실행 시 마지막 성공 결과가 동일해야 하므로 편성은 현재 시각의 KST 날짜와 그날의 `baseline` 상태를 기준으로 계산한다.

핀 저장 후 어드민은 저장 시작 시각보다 `home_dynamic/current.updated_at`이 새로워질 때까지 최대 15초간 확인한다. 갱신되면 최종 미리보기를 새로 읽고, 갱신되지 않으면 `핀은 저장됨 · 편성 갱신 대기`로 표시한다. 트리거가 실패해도 다음 일간 실행에서는 저장된 핀이 다시 반영된다.

## 8. 어드민 UX

기존 헤더의 `NEW 추천 롤링 베너`를 `홈 상단 관리`로 바꾸고, 기존 `/edit-main-book` 화면을 `home_carousel_pins` 관리 화면으로 대체한다.

### 제공 기능

- 제목 검색으로 `books`에서 책 선택
- 활성 여부
- KST 기준 노출 시작일과 종료일(선택, 양 끝 날짜 포함)
- 최종 위치 지정
- 위/아래 이동 버튼으로 위치 변경
- 핀 수정·삭제
- 활성 핀과 자동 5권을 합친 현재 `home_dynamic/current.carousel` 미리보기
- 자동 선정 결과는 읽기 전용으로 표시

### 검증

- 존재하지 않는 책 저장 금지
- 숨김 책 저장 금지
- 같은 책의 활성 핀 중복 금지
- 같은 위치의 활성 핀 중복 금지
- 시작일이 종료일 이후인 값 금지
- 저장·삭제 후 편성 갱신 완료 여부 표시

기존 `main_books` 데이터는 수정하거나 삭제하지 않는다. 이번 출시 후에는 어드민에서 `main_books`를 편집하지 않으며, 필요 시 Firestore에서 별도로 복구·관리한다.

## 9. 오류 처리와 폴백

- 자동 역할 일부가 비면 공통 보충 규칙으로 가능한 한 5권을 채운다.
- 핀의 책이 삭제됐거나 숨김 상태면 그 핀은 최종 배열에서 건너뛰고 경고 로그를 남긴다.
- 모든 자동 후보 계산이 실패하면 `home_dynamic/current`를 덮어쓰지 않는다.
- 핀만 유효하고 자동 후보가 부족하면 유효 핀과 가능한 자동책만 기록한다.
- `home_dynamic/current.carousel`이 비거나 읽기 실패하면 현재 모바일 앱의 `main_books` 폴백이 그대로 작동한다.

## 10. 테스트

### 순수 함수 단위 테스트

- 역할별 한 권 선정과 역할 간 중복 제거
- `trending`, `discover`, 핀 제외
- 14일 노출 제한과 후보 부족 시 완화
- KST 날짜별 결정적 순환 및 같은 날 멱등성
- 핀 위치 1부터 시작하는 조합
- 두 핀 사이의 빈 슬롯 자동 채우기
- 중복 위치·범위 초과·중복 책 서버 보정
- 핀 N권과 자동 5권이 최종 `N+5`권이 되는지 확인

### 생성기 통합 테스트

- `main_books`가 새 캐러셀 선정에 영향을 주지 않는지 확인
- `home_dynamic/current`의 기존 shape 유지
- `_carousel_state` 갱신과 오래된 상태 정리
- 핀 생성·수정·삭제 후 재생성 결과
- 실패 시 직전 `home_dynamic/current` 보존

### 어드민 검증

- 프로덕션 빌드 성공
- 책 검색, 핀 추가·수정·삭제
- 위치 이동 및 충돌 차단
- 기간별 활성 상태 표시
- 최종 캐러셀 미리보기 순서 확인

## 11. 배포와 검증

1. Functions 테스트와 lint를 실행한다.
2. Vue 어드민을 빌드한다.
3. Functions를 배포한다.
4. 핀이 없는 상태에서 수동 재생성하고 자동 5권 및 전역 중복 제거를 확인한다.
5. 테스트 핀을 위치 1과 4에 추가해 기존 동적 앱에서 최종 순서를 확인한다.
6. 아주 오래된 앱 또는 Firestore 직접 조회로 `main_books`가 변하지 않았음을 확인한다.
7. 확인 후 어드민 hosting을 배포한다.

모바일 앱 배포는 필요하지 않다. 기존 앱에서는 선정 역할 이름이나 이유 문구가 보이지 않고 표지 캐러셀만 새 배열대로 표시된다.

## 12. 성공 관찰 지표

현재 모바일 앱 변경 없이 활용 가능한 지표만 사용한다.
