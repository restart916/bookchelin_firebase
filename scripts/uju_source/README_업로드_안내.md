이 폴더(`handoff/`)를 통째로 프로젝트에 넣고, 아래 "클로드 코드에게 줄 요청"을 그대로 전달하면 됩니다.

## 2. 폴더 안의 파일

```
handoff/
├─ jamdeulgi-jeon-uju.epub      ← 완성된 EPUB (전자책 통파일)
├─ build_epub.py                ← 마크다운 → EPUB 재빌드 스크립트
└─ content/
   ├─ uju-00-series-guide.md    ← 시리즈 구성/표준 템플릿 (메타 자산, 업로드 대상 아님)
   ├─ uju-01-byeolui-ilsaeng-v2.md
   ├─ uju-02-eunha.md
   ├─ ... (3~9화)
   └─ uju-10-kkeut.md
```

- **EPUB 단위로 올리는 서비스라면** → `jamdeulgi-jeon-uju.epub` 하나만 쓰면 됨.
- **화(챕터) 단위로 DB에 넣는 서비스라면** → `content/uju-01 ~ uju-10` 마크다운 10개를 순서대로 넣고, 각 파일 상단 YAML(메타데이터)을 그대로 필드에 매핑.

각 마크다운 파일 맨 위에는 아래 같은 메타데이터 블록이 있음:
```yaml
series: "잠들기 전 우주"
episode: 2
book_total_episodes: 10
title: "은하라는 도시"
length_label: "10분"
read_minutes: 10
difficulty: "입문"
tags: [...]
audio_ready: true        # TTS 낭독 가능하게 작성됨
autoscroll_ready: true   # 자동 스크롤 대응
source_note: "..."       # 저작권 출처 표기
```

## 3. 클로드 코드가 먼저 확인해야 할 것 (중요)

나(이전 세션)는 북슐랭 코드베이스의 실제 구조를 모릅니다. 그래서 클로드 코드가 **코드를 보고 직접 확인/결정**해야 하는 항목입니다:

1. **업로드 단위**: 북슐랭이 책을 "EPUB 통파일"로 받는지, "책 → 챕터" 구조의 DB 레코드로 받는지. → 스키마/모델 파일 확인.
2. **데이터 모델 매핑**: 위 YAML 필드(title, episode, tags, read_minutes 등)가 실제 DB 컬럼/모델의 어떤 필드에 대응하는지.
3. **페이지 기반 정산 모델**: 북슐랭은 "읽은 페이지 기반" 수익 배분 구조. 이 자체 제작 책은 외부 출판사 분배가 없으므로, 정산 대상에서 제외하거나 "자체 콘텐츠" 플래그로 분류해야 하는지 확인. (출판사 정산에 잘못 섞이면 안 됨.)
4. **업로드 방식**: 관리자 페이지 폼인지, CLI/스크립트인지, API 엔드포인트인지. 기존에 책을 어떻게 넣었는지 코드/스크립트 확인.
5. **표지 이미지**: 현재 EPUB은 텍스트 표지만 있음. 썸네일/커버 이미지가 별도로 필요한 서비스면 커버 이미지 1장을 따로 만들어야 함(아래 7번 참고).

## 4. 클로드 코드에게 줄 요청 (복붙용)

> handoff/ 폴더에 "잠들기 전 우주"라는 자체 제작 교양서(10화)가 있어.
> 먼저 우리 코드베이스에서 책/챕터가 어떤 모델·스키마로 저장되고, 기존 책은 어떤 방식(관리자 폼/스크립트/API)으로 업로드되는지 확인해줘.
> 그다음 이 책을 그 방식에 맞게 업로드하는 작업을 해줘.
> - 업로드 단위가 EPUB이면 jamdeulgi-jeon-uju.epub 사용
> - 챕터 단위면 content/uju-01~uju-10 마크다운을 순서대로 넣고, 각 파일 상단 YAML을 해당 필드에 매핑
> - 이건 외부 출판사 책이 아니라 자체 콘텐츠니까, 페이지 기반 정산에서 출판사 분배 대상이 되지 않도록 분류/플래그를 확인해줘
> 진행 전에, 위에서 확인한 모델 구조랑 업로드 계획을 먼저 요약해서 보여주고 시작해줘.

## 5. 저작권 / 안전 메모

- 본문은 **일반 천문 상식을 자체 재구성**한 글이라 특정 도서·영상을 베끼지 않았고, 사실(fact)에는 저작권이 없어 안전지대에 있음. 각 화 `source_note`에 그 취지를 명시해 둠.
- 본문에 외부 이미지는 **하나도 포함돼 있지 않음**(EPUB 빌드 시 이미지 슬롯 주석은 자동 제거됨). 따라서 현재 상태로는 이미지 저작권 리스크 없음.
- 나중에 이미지를 넣을 경우 **퍼블릭 도메인(NASA) 또는 CC0(Unsplash/Pexels/Pixabay)만** 사용하고 캡션에 출처 표기. 일반 웹 이미지 사용 금지.

---

## 6. 출판 전 권장 검수

- 본문은 입문 교양 수준이라 큰 오류는 없지만, **숫자·연대(예: 사십육억 년, 백삼십팔억 년 등) 사실관계를 한 번 검수** 권장.
- TTS 낭독을 전제로 숫자를 한글로 풀어 씀("46억"→"사십육억"). 화면 표시용으로 숫자 표기가 더 낫다면 별도 표시 버전을 둘 수 있음.

## 8. 재빌드 방법 (참고)

마크다운을 수정한 뒤 EPUB을 다시 만들려면:
```bash
pip install ebooklib --break-system-packages
python3 build_epub.py
```
`build_epub.py`는 `content/` 안의 uju-01(v2)~uju-10 마크다운을 읽어 표지·차례·10화를 묶어 EPUB을 생성함. 이미지 슬롯 주석은 빌드 시 자동 제거됨.
